## UI design rules

Build interfaces with a clean modern SaaS aesthetic.

Do:
- Use a white or near-white app background.
- Use subtle borders, soft shadows, rounded corners, and generous spacing.
- Prefer neutral palettes: slate, zinc, stone, white.
- Use one restrained accent colour.
- Use clear hierarchy: page title, description, cards, tables, primary actions.
- Make layouts feel like Linear, Stripe, Vercel, Notion, or modern B2B SaaS.

Do not:
- Use purple/blue/pink gradient backgrounds.
- Use glassmorphism unless explicitly requested.
- Use oversized emoji, novelty colours, or landing-page gradients inside the app.
- Add decorative background blobs.
- Make everything look like a generic AI demo.

Default layout:
- App shell with sidebar or top nav.
- Main content max width around 1200px.
- Cards with `border`, `bg-white`, `shadow-sm`.
- Background: `bg-slate-50` or `bg-white`.
